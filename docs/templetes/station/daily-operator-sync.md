`station/ops/daily-operator-sync.md`

---

# Daily Operator Sync

**Date:**
**Operator:** Rajah
**Session ID:**

---

# 1. Check-In Round

Status

> How are you mentally and physically?

Focus

> What area deserves your energy today?

Constraint

> What might limit your capacity today?

Example format:

```
Status: steady, rested
Focus: DU library routing and document intake
Constraint: limited time window
```

---

# 2. AAR — Previous Session

What Happened

> Brief recap of the last working session

What Went Well

> Progress, breakthroughs, stable systems

What Was Difficult

> Friction points, confusion, blockers

What We Learned

> Lessons to carry forward

Example:

```
What Happened:
Explored station documentation tree and intake flow.

What Went Well:
Structure is clearer. Tree layout supports harvesting.

What Was Difficult:
Sorting model still missing.

What We Learned:
Library intake + placement system must be solved first.
```

---

# 3. Mission Board

## 🟦 System Build

Small technical objectives for today.

```
• Review station tree
• Test document intake parser
• Outline library routing model
```

## 🟩 Job Hunt

Career progress tasks.

```
• Review job listings
• Save promising roles
• Draft one application
```

## 🟨 Life Ops

Personal maintenance tasks.

```
• Breakfast
• Short walk
• Reset break
```

---

# 4. Execution Window

Start Time:
Estimated Work Window:

Primary Target:

> The single most important objective today

---

# 5. Closing Marker

When the session ends:

```
Mission status: Complete / Partial / Aborted
Notes:
Next Focus:
```

---

# Optional Tag System (helps later)

You could add tags to each entry:

```
#sync
#aar
#mission-board
#station-build
#job-hunt
```

Later you could **search or parse them automatically** if you build a tool around your station logs.
