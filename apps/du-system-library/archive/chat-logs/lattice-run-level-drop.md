## FAQ

**Q: Do I log every single playtest?**    
A: Only if it's canonical (real story progression) or reveals something significant. Quick bug tests can be batch-logged.

**Q: What if a run is partially complete?**    
A: Mark `Session Status: 🔄 In Progress` and update when finished. Add `[CONTINUED]` to title.

**Q: Can I split a long run into multiple entries?**    
A: Yes—use sequential IDs (001, 002) and link them in Cross-References.

**Q: What if I forgot to log a run?**    
A: Write it from memory/notes ASAP. Mark `[RETROACTIVE LOG]` in header. Better late than never.

**Q: Should I log failed/abandoned runs?**    
A: Yes if they had story beats or creative output. No if it was just "crashed immediately."

**Q: How detailed should Story Beats be?**    
A: Detailed enough that someone else could understand what happened and why it mattered. Aim for 3–5 sentences per beat.

**Q: What counts as "Creative Output"?**    
A: Songs, images, lore text, dialogue, flavor text, even memes/jokes that capture the session vibe.
