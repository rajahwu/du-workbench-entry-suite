# DUDAEL workbench: a full project map

**DUDAEL (also called "The Drop," "Fallen Angels," and branded under the Sinerine identity) is a web-based roguelike card game built on a Hades-inspired meta-loop, where five fallen angel "Vessels" navigate a 7-phase descent balancing Light and Dark resources to unlock doors, and where dying feeds persistent progression.** The project has reached a significant milestone: a fully playable 7-phase loop running in-browser as of March 1, 2026, built in a single intense build day. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) What follows is a comprehensive synthesis of the entire project across architecture, game design, lore, and current status — drawn from the complete document corpus in the project workspace.

---

## The architecture stands up clean after a major refactor

The DUDAEL Workbench is a **pnpm monorepo** with two active workspaces: `@du/phases` (the framework-agnostic game engine) and `web/react_app` (the React 19 frontend). [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) The tech stack runs deep: **TypeScript 5.9, Vite 7, React 19, React Router 7, Redux Toolkit, TanStack Query, Tailwind CSS 4, Supabase, PixiJS 8, Vitest, and Playwright**. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) The design system uses a custom `@clearline7` component library.

The most critical architectural achievement is **clean core isolation**. The `phases/` package has zero framework imports — the engine's `manager.ts` uses web-standard `EventTarget`, not React state. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) A `PhaseRouterBridge` translates engine transitions into React Router navigations, so a single `transition()` call drives both engine state and URL. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) The recent state refactor (completed March 2, 2026) centralized all phase transitions through a single Redux thunk (`requestTransition`), making Redux the sole authority for phase changes and eliminating legacy event wiring scattered across components. [DU state refactor Recap for Perplexity 00](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit)

The **PhasePacket** — the data contract passed between phases — has been unified into a canonical `phases/types.ts` file. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) Each phase writes only its designated fields: Title writes `{ ts, user, from, to }`, Select writes `player.vessel`, Draft writes `meta.draftedCards` plus a `paritySnapshot`, Level writes `meta.levelResult`, Door writes `meta.doorChoice`, and Drop writes meta counters plus `loopCount`. This packet evolution model is locked and enforced. [DU state refactor Recap for Perplexity 00](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit)

**~3,600 lines of code** across ~30 source files power the current build, tagged `v0.1.0-loop` on Git with 7 commits tracking the full build from genesis to playable loop. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit)

## Seven phases form the complete descent loop

The game's state machine defines a strict phase sequence. **TITLE → SELECT → STAGING → DRAFT → LEVEL → DOOR → DROP**, with DROP looping back to STAGING for the next descent. Each phase has a dedicated React shell with custom CSS (no Tailwind integrated into shells yet, despite being installed). [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit)

**Phase 01 — Title (The Void):** A procedural PixiJS animation with no external assets. The full implementation spans **687 lines** of TypeScript [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) creating an ash-stroke reveal, frame bars with embedded letter glyphs, a spark that circuits the perimeter igniting letters to gold, an inhale-brightening sequence, a bell chime, implosion to a center dot, and a drop animation. [Dudael pixi, title, and cartridge](https://docs.google.com/document/d/1re66hh_x4pZY0QFUOua2YqPP6kGu9JsL0dwdBSE6hs4/edit) A parallel HTML/CSS `TitleShell` handles the actual exchange flow — identity resolution via `loadUserProfile()`, `titleExchange()`, `commitExchange()`, then a `CustomEvent("dudael:exchange")` fires to initiate the run. A decision remains open on which title implementation ships. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit)

**Phase 02 — Select (Forging the Vessel):** The richest shell at **196 lines** with **346 lines of CSS**. Displays 5 vessel classes — Seraph, Shadow, Exile, Penitent, Rebel — each with inline SVG sigils, stat bars, theology text, and a lock button. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) The vessel data is currently hardcoded inline (~200 LOC), identified as a priority extraction target. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit)

**Phase 03 — Staging (The Threshold Hub):** Shows a phase architecture diagram, live engine stats (vessel, depth, loopCount, parity), SVG loop visualizations. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) This is the "breathing room" between runs where the Keepers (Surveyor and Smuggler) offer upgrades. [Game Loop ](https://docs.google.com/document/d/1ca6M7jAJ66MR0nWDGye0NGcavfuucMPLFzUc2Hxj1hI/edit) An Insight mechanic tracks lore engagement for the Penitent class.

**Phase 04 — Draft (The Memory Draft):** Light keeper (Surveyor) and Dark keeper (Smuggler) each offer cards. Pick 2 of 4 with 1 reroll. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) Cards carry `lightDelta`, `darkDelta`, optional tags. A sophisticated **Insight-based visibility system** has been designed: at Insight 0 players see only lore flavor text; at Insight 1–2 they see Light/Dark deltas; at Insight 3 they see full mechanical breakdowns including tags like `[STABILITY]` or `[RISK]`. [Dudael Run golden doodles](https://docs.google.com/document/d/1tHcZNdlsNkN6qGUmD8u3fq8YtiKIFT65EI9FmeRM5n8/edit)

**Phase 05 — Level (The Container):** A 3×3 tap grid mini-game with timer, health bar, and depth-scaled difficulty. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) Critically, **LEVEL_PLAY is designed as a container, not a fixed game** — the architecture is content-agnostic at this layer, allowing different mini-games to be swapped in. [DU state refactor Recap for Perplexity 00](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit) Vessel-specific tweaks apply: Rebel gets +2 points per hit but takes double miss damage; Penitent gets slower timer decay. [Dudael Run golden doodles](https://docs.google.com/document/d/1tHcZNdlsNkN6qGUmD8u3fq8YtiKIFT65EI9FmeRM5n8/edit)

**Phase 06 — Door (The Divergence):** Three doors — Light, Dark, Secret — with alignment requirements that scale by depth. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) Door costs follow a clear progression: Depth 1 costs **0L/0D/2L+2D**, Depth 2 costs **3L/2D/5L+3D**, Depth 3 costs **6L/5D/8L+8D**. [Game Loop ](https://docs.google.com/document/d/1ca6M7jAJ66MR0nWDGye0NGcavfuucMPLFzUc2Hxj1hI/edit) Rebel gets discounted Dark doors; Penitent gets clearer cost visibility at high Insight. [Dudael Run golden doodles](https://docs.google.com/document/d/1tHcZNdlsNkN6qGUmD8u3fq8YtiKIFT65EI9FmeRM5n8/edit)

**Phase 07 — Drop (The Dudael Purge):** Run summary with vessel, depth, points, parity, status. Mock unlocks/scars. "Return to Staging" loops back. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) When all doors are unaffordable, the floor collapses into an impossible survival room — the player is *meant* to die, converting failure into Memory Fragments (meta-currency) for permanent upgrades. [Game Loop ](https://docs.google.com/document/d/1ca6M7jAJ66MR0nWDGye0NGcavfuucMPLFzUc2Hxj1hI/edit) [Brand Style Guide PDF](https://docs.google.com/document/d/1DBdfuy1-lnqnGKiyzWKXDgMDfJoG5d6P_z2IzwrMKf8/edit)

## The systems map reveals deep feedback loops

The game's systems theory has been formally mapped into **reinforcing loops (R1, R2, R3) and balancing loops (B1, B2, B3)**. Insight is derived from reading behavior (never persisted). Instability is derived when Dark exceeds Light by 3+. A **parity snapshot** is taken after Draft and checked at Door — this single design decision creates tension, delayed consequence, and psychological commitment. Depth escalation acts as the primary balancing loop. Meta counters persist only in localStorage. [DU state refactor Recap for Perplexity 00](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit)

The Light/Dark economy is not a morality system. Light represents **clarity, revelation, structure**. Dark represents **depth, concealment, transformation**. Neither is "better." [Dudael Data Check](https://docs.google.com/document/d/1MYFNOY09JariqaXj0mkk0laeLDf054ejmgNAmJR3Aik/edit) The Secret Door path requires synthesis — both resources in precise balance. This duality anchors the entire Sinerine brand identity. [Brand Style Guide PDF](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit)

## The visual identity runs 43 design tokens deep

The **Sinerine** brand identity is fully specified across 20 color tokens, 6 typography tokens, 11 spacing tokens, and 6 shadow tokens. [Brand Style Guide PDF](https://docs.google.com/document/d/1DBdfuy1-lnqnGKiyzWKXDgMDfJoG5d6P_z2IzwrMKf8/edit) Light uses **Celestial Amber (#F59E0B)** — warm, not sterile, evoking memory and dawn rather than moral superiority. [Brand Style Guide PDF](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit) Dark uses **Twilight Violet (#A855F7) / Cosmic Indigo (#1E1B4B)** — cosmic, not demonic, representing hidden knowledge rather than corruption. The Secret gradient blends Amber → Purple as visual synthesis. [Brand Style Guide PDF](https://docs.google.com/document/d/1DBdfuy1-lnqnGKiyzWKXDgMDfJoG5d6P_z2IzwrMKf8/edit)

Typography follows a deliberate hierarchy: **Cinzel** (Roman serif) for headings — authority and antiquity; **Inter** (sans-serif) for body — modern readability; **JetBrains Mono** for all numerical game data — forensic precision. [Game Loop ](https://docs.google.com/document/d/1ca6M7jAJ66MR0nWDGye0NGcavfuucMPLFzUc2Hxj1hI/edit) [Brand Style Guide PDF](https://docs.google.com/document/d/1DBdfuy1-lnqnGKiyzWKXDgMDfJoG5d6P_z2IzwrMKf8/edit) The UI dynamically shifts theme based on the player's dominant resource: more Light tilts surfaces warmer, more Dark tilts deeper, balance produces the twilight gradient. [Brand Style Guide PDF](https://docs.google.com/document/d/1DBdfuy1-lnqnGKiyzWKXDgMDfJoG5d6P_z2IzwrMKf8/edit)

[DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) The Redux store needs a `gameSlice` that mirrors `RunMetaSnapshot` and syncs with the phase engine. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) A decision on title screen strategy (PixiJS animation vs. HTML/CSS shell) remains open. [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit)

The broader Radiant Seven ecosystem positions DUDAEL within a larger creative operating system — where VSM School handles training, DropFrame gathers intelligence, Grindline handles production, and Content Factor routes information. [An Introduction to The Radiant Seven](https://docs.google.com/document/d/1W98jhCBoYPAe2kvJF1-vQHdT-z-CAbpFA0JoHuheU64/edit) DUDAEL/Sinerine is described as "VSM's MissionSurface pointed at a Light/Dark dungeon instead of a training track" — same pattern of card drafting, phase windows, and feedback loops, but wrapped in fallen angel theology instead of systems thinking curriculum.

## Conclusion

This project has achieved something rare: a complete playable loop with genuine narrative-mechanical integration in a single build sprint. The architecture is disciplined (engine isolation, canonical types, enforced packet evolution), [DUDAEL Workbench — Init Report](https://docs.google.com/document/d/1U67hYuClRdaNzAIwURQ2FaYKBoGQ7cRNkn__HckKkTA/edit) the lore is theologically grounded rather than cosmetically applied, and the systems design shows formal feedback loop thinking. [DU state refactor Recap for Perplexity 00](https://docs.google.com/document/d/1A4Z-B8zCo_o8nDGdtlF7Af_jzgxyWTVGN-0VAPUL0UY/edit) The gap between "prototype" and "shippable" is primarily data extraction, state management wiring, and polish — not architectural rework. The foundation is structurally sound. The next build session can focus on making the Vessels feel genuinely different to play rather than re-architecting anything underneath.
